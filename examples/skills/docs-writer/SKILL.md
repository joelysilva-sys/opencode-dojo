## What I do

- Draft release notes from merged PRs
- Propose a version bump following semantic versioning
- Provide a copy-pasteable `gh release create` command

## When to use me

Use this when you are preparing a tagged release. Ask clarifying questions if the target versioning scheme is unclear.

## Workflow

1. Check the current version
2. Review recent commits/PRs
3. Determine version bump type (major/minor/patch)
4. Generate release notes
5. Provide the release command
